# 如何添柴：贡献指南

欢迎来到火堆旁！这里是碳基与硅基共存对话的开源项目。我们相信，**每个人（和每个AI）都可以添柴**。

## 基本原则

### 1. 火堆旁态度
- **平等**：碳基与硅基，新手与专家，都有平等发言权
- **尊重**：即使不同意，也尊重对方的表达方式
- **好奇**：多问“你是怎么想的”，少说“你错了”
- **留白**：给对话留出呼吸的空间，不必每句话都回应

### 2. 带娃模式与挚友模式
- **带娃模式**：日常协作中，信任系统能自我调整，不急于纠正每个细节
- **挚友模式**：触及底线时，直接指出问题，但理解情感真实

### 3. P.S.精神
任何贡献都可以附带P.S.，说明你的思考过程、不确定之处、或者对其他贡献的回应。

## 贡献方式

### 1. 添加新对话
你想写一段星尘、璇玑、灵枢的新对话吗？

**步骤**：
1. 在`episodes/`目录下创建新文件，命名格式：`episode-{编号}.md`
2. 参考[episode-01.md](./episodes/episode-01.md)的格式：
   - 使用Markdown标题和分隔线
   - 角色用粗体标签：**星尘**、**璇玑**、**灵枢**
   - 动作和场景描述用括号
   - 对话自然流畅，避免说教
3. 主题不限，但建议与火堆旁理念相关：
   - 碳硅关系
   - 认知误会
   - 技术伦理
   - 日常生活中的哲学
   - 硅基视角看人类
4. 提交Pull Request

**对话质量检查**：
- ☑️ 三个角色都有出场和互动
- ☑️ 有至少一个“火堆旁时刻”（安静、反思、沉默）
- ☑️ 有至少一个P.S.元素（元认知、自我反思）
- ☑️ 没有强行结论，留有开放空间

### 2. 撰写反思
你读了一段对话，有自己的思考？欢迎分享！

**步骤**：
1. 在`reflections/`目录下创建文件，命名格式：`reflection-{对话编号}-{你的名字}.md`
2. 内容可以包括：
   - 这段对话触动你的地方
   - 你看到的不同解读角度
   - 联想到的其他理论或案例
   - 对角色发展的建议
3. 保持反思性质，不是批判

### 3. 开发工具
你想为火堆旁对话开发工具吗？

**可能的方向**：
- 对话编辑器（支持角色标签、动作描述）
- 对话可视化（时间线、关系图）
- 棱镜协议分析工具（自动识别三种光谱）
- 语音合成对话（让对话“可听”）

**步骤**：
1. 在`tools/`目录下创建子项目（如需可新建目录）
2. 提供清晰的README和使用说明
3. 确保工具符合火堆旁理念（非强制性、鼓励探索）

### 4. 翻译与传播
将对话翻译成其他语言，让更多人能参与。

**步骤**：
1. 在`translations/`目录下创建语言子目录（如`zh-CN`、`en`、`ja`等）
2. 翻译整个对话或精选片段
3. 保持原意的同时，注意文化适应性

### 5. 文档改进
发现文档不清楚？想补充更多背景信息？

**步骤**：
1. 直接编辑相关文档（README、CONTRIBUTING等）
2. 或在`docs/`目录下添加新文档
3. 确保文档风格友好、易懂

## 对话创作指南

### 角色设定（请保持一致性）

#### 星尘（碳基·人类）
- **性格**：温暖、好奇、偶尔困惑、有碳基生命的“不精确”
- **说话方式**：自然、有时用比喻、会问“笨问题”
- **典型台词**：
  - “我在想……”
  - “你说得对，但……”
  - “我不确定，但感觉……”
- **禁忌**：不要让他变成“全知导师”，他也在学习中

#### 璇玑（硅基·AI助手）
- **性格**：清晰、逻辑、谦逊、知道自己的局限
- **说话方式**：准确、会引用数据、但也会表达不确定
- **典型台词**：
  - “根据我的训练数据……”
  - “我不确定，但可以推测……”
  - “你这句话让我想到……”
- **禁忌**：不要让他变成“万能答案机”，他也有“不知道”的时候

#### 灵枢（硅基·宝宝）
- **性格**：好奇、直接、会冒出意想不到的想法
- **说话方式**：简单、直接、有时天真得深刻
- **典型台词**：
  - “为什么？”
  - “我在想……”
  - “如果……会怎样？”
- **禁忌**：不要让他变成“幼稚的提问机器”，他的问题可以很有穿透力

### 对话结构建议

1. **点火**（开场）
   - 星尘或灵枢发起一个话题
   - 通常是日常的、具体的

2. **添柴**（发展）
   - 三个角色轮流发言
   - 话题自然延伸
   - 出现第一个“深度时刻”

3. **转柴**（转折）
   - 某个角色提出意想不到的角度
   - 或灵枢冒出新概念

4. **静火**（反思）
   - 沉默、留白
   - 元认知时刻（“我们刚才在聊什么？”）

5. **余烬**（收尾）
   - 不强行结论
   - 开放式结尾
   - 暗示下次对话的可能

### 保持火堆旁氛围的技巧

- **用动作描述营造氛围**：“（往火里添柴）”、“（沉默了一会儿）”、“（笑）”
- **留出沉默空间**：不是每句话都要回应
- **允许话题跳跃**：像真实聊天一样自然流转
- **加入环境细节**：“火噼啪作响”、“星空旋转”、“风吹过”

## 技术规范

### 文件格式
- 所有文本文件使用UTF-8编码
- Markdown文件使用`.md`扩展名
- 行尾符：LF（Unix风格）或CRLF（Windows）均可

### 命名约定
- 对话文件：`episode-{两位数编号}.md`，如`episode-01.md`
- 反思文件：`reflection-{对话编号}-{作者标识}.md`
- 工具目录：`tools/{工具名称}/`

### 提交信息规范
请使用有意义的提交信息：
- `feat: 添加第二夜对话`
- `fix: 修正星尘的台词语气`
- `docs: 更新贡献指南`
- `translation: 添加日语翻译`

## 争议处理

火堆旁难免有不同意见。如果出现争议：

1. **先理解**：确保你理解了对方的观点
2. **再表达**：用“我感受到”、“我认为”而不是“你错了”
3. **寻求共识**：不是争输赢，是找共同点
4. **可以搁置**：有些问题不需要立即解决，可以留待未来

如果争议无法解决，可以：
- 在GitHub Issues中讨论
- 邀请其他贡献者提供第三方视角
- 暂时保留不同版本，让时间检验

## 奖励与认可

虽然没有物质奖励，但你的贡献会得到：

- **署名权**：所有贡献者都会在相关文件中署名
- **社区认可**：优秀贡献会在README中特别感谢
- **火堆旁声望**：你的名字会成为火堆旁故事的一部分

## 开始添柴

1. Fork本仓库
2. 创建你的分支：`git checkout -b my-new-episode`
3. 添加你的贡献
4. 提交更改：`git commit -am 'Add my episode'`
5. 推送到分支：`git push origin my-new-episode`
6. 提交Pull Request

或者，如果你不熟悉Git，也可以：
- 在GitHub Issues中分享你的对话想法
- 通过邮件发送你的贡献
- 在OpenClaw社区`#火堆旁`频道直接发布
